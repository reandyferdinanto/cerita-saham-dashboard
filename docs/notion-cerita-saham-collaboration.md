# Cerita Saham

## 1. Gambaran Besar

### Nama Produk
Cerita Saham

### Inti Produk
Cerita Saham adalah platform edukasi dan workflow trading/investing yang membantu user membaca pasar saham Indonesia dengan cara yang lebih praktis, terstruktur, dan mudah dipahami.

Platform ini tidak hanya menampilkan data, tapi membantu user:

- memahami apa yang sedang terjadi di market
- menemukan saham yang layak diperhatikan
- menyimpan watchlist dan rencana trading
- membaca sinyal teknikal dan bandarmology dengan bahasa yang lebih sederhana
- belajar membangun proses, bukan sekadar ikut sinyal

### Value Utama
- Mengubah data pasar menjadi insight yang lebih mudah dicerna
- Menjadi jembatan antara edukasi pemula dan workflow trader/investor yang lebih serius
- Membantu user disiplin lewat tools, catatan, dan struktur berpikir

---

## 2. Konsep Dasar Produk

### Masalah yang Ingin Diselesaikan
Banyak user retail:

- bingung membaca market
- tidak tahu saham mana yang layak dipantau
- masuk saham tanpa rencana TP/SL
- sulit membedakan edukasi, opini, dan keputusan eksekusi
- punya data banyak, tapi workflow-nya tidak rapi

### Solusi Cerita Saham
Cerita Saham dibangun sebagai kombinasi:

- dashboard market
- tools investor/trader
- watchlist berbasis workflow
- edukasi yang langsung nyambung ke penggunaan produk
- admin control center untuk mengelola member, artikel, watchlist, dan analisa

### Positioning
Bukan sekadar portal berita saham.  
Bukan sekadar stock screener.  
Bukan sekadar komunitas premium.

Cerita Saham diposisikan sebagai:

> platform pendamping keputusan untuk user saham Indonesia, dengan pendekatan edukatif, sistematis, dan praktis.

## 7. Alur Web Cerita Saham Secara Teknis

### Penjelasan Sederhana
Secara teknis, web ini bekerja seperti sistem yang terdiri dari 4 lapisan:

#### 1. Tampilan Depan
Ini yang dilihat user di browser:

- halaman dashboard
- halaman login
- halaman watchlist
- halaman admin
- halaman tools

Tampilan dibuat dengan Next.js + React.

#### 2. API / Otak Aplikasi
Saat user klik sesuatu, frontend memanggil API internal, misalnya:

- ambil data user
- ambil watchlist
- simpan portfolio
- upload stock summary
- update membership

API ini ada di folder `app/api/...`

#### 3. Business Logic / Proses Inti
Di layer ini aplikasi memproses:

- validasi login
- perhitungan risk calculator
- analisa technical signal
- bandarmology scoring
- parsing file IDX
- pengaturan akses member

Logic ini banyak ada di folder `lib/`

#### 4. Database / Tempat Simpan Data
Sekarang aplikasi mendukung dua jalur:

- MongoDB
- Postgres

Dengan mode:

- `mongo`
- `postgres`
- `postgres-prefer`

Artinya, aplikasi bisa:

- tetap jalan pakai Mongo
- pindah ke Postgres
- atau pakai Postgres lebih dulu sambil tetap punya fallback Mongo

## 9. Alur Data Teknis yang Mudah Dipahami

### Contoh: User Login
1. User isi email + nomor telepon
2. API login mengecek data user
3. Jika cocok, sistem membuat token login
4. Token disimpan di cookie
5. Cookie ini dipakai untuk tahu siapa user yang sedang aktif

### Contoh: User Membuka Watchlist
1. Halaman watchlist request ke API `/api/watchlist`
2. API mengambil data dari repository
3. Repository memilih database berdasarkan `DB_PROVIDER`
4. Data dikirim balik ke browser
5. Frontend menampilkan saham, TP, SL, dan catatan

### Contoh: Admin Upload Stock Summary
1. Admin upload file IDX
2. API membaca file Excel
3. Sistem parse header dan isi file
4. Data disimpan ke database
5. Data dipakai untuk analisa akumulasi dan chart

### Contoh: Membership Check
1. User akses halaman premium
2. `proxy.ts` membaca cookie login
3. Sistem cek role dan membership status
4. Jika aktif, user boleh masuk
5. Jika belum aktif / expired, user diarahkan ke halaman pending

- frontend, API, dan logic cukup rapi dipisah

## 11. Usulan Narasi Singkat untuk Kolaborator

Cerita Saham adalah platform yang menggabungkan edukasi, market dashboard, watchlist, dan tools keputusan saham Indonesia dalam satu workflow yang lebih mudah dipahami. Tujuan utamanya bukan hanya memberi data, tapi membantu user berpikir lebih rapi saat membaca market dan mengambil keputusan. Secara bisnis, platform ini bisa tumbuh lewat membership bertingkat. Secara teknis, sistem sudah punya admin control center, tools investor, dan mulai mendukung arsitektur database yang lebih fleksibel untuk scale di VPS.
